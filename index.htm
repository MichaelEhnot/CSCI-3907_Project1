This project combines three color-filtered grayscale images into one color image by aligning them on top of each other and assigning the appropriate color to each image. Similar to how 3 channel images are used today.

 

This is the original image. There are three separate images with a different color filter applied to each one.

 

When you lay the images on top of each other and assign RGB channels, the colors are shown. This image is just each panel shifted up to the top one with no dynamic alignment. The goal of the project was to algorithmically align the panels

 

Single Scale

The first alignment algorithm is a simple single scale method. This algorithm tries every combination of two panels in a 30-pixel range in the x and y directions and scores each alignment based off the sum of squared differences of the panels. The best scoring alignment is kept and returned from the function. This is done twice. The first alignment aligns the second panel with the first and the second alignment aligns the third panel with the first. Below are some of the before and after images from the single scale algorithm. Before is on the left and after is on the right.

Some look very good, and others only have a small improvement

 

Multi-Scale

The multi scale approach is for larger images where it takes a long time to compute the sum of squared differences. This algorithm starts by using the single scale alignment on a 25% resolution version of the image with a range of 20 pixels. This part can be computed quickly because the image is much smaller than the original. Then, another single scale alignment is done on a 50% resolution version of the image with a smaller range of 10 pixels and the previous alignment pre-applied before this alignment begins. Finally, a 5 pixel single-scale alignment is done on the full image with the previous two alignments pre-applied to the image. This method allows for a large range of alignments to be tested on a large image where the alignment between panels can be many pixels away.

The multi scale alignment was used on images around 7-9 MB and a few small images for testing purposes. This algorithm would complete a large image in about a minute where the single scale algorithm would take too long.

Similar results as the single scale, but multi-scale was actually able to process large images in reasonable time

 

Border Removal

As an extra feature, border removal was done on the images. A lot of the images had extra white borders around them, and the border removal changes those white borders into black borders that fit in with the image better. The algorithm analyzes a row to the image and for every pixel that is completely white, the whole column is turned black. This can be seen on many of the images above. You can tell where the artificial border ends and the image begins because there is usually a pure black area with the slightly off-white portion of the image next to it. The filter worked well on most images, but a few were messed up by the filter.

Reflection

For the most part, all of the images were improved by both of the alignments. There are definitely still some images that look like they can be improved. My first thought about improving some would be to focus of the subject of the images. A lot of the landscape images look good because there is not one particular thing to look at, but a lot of the pictures with people or staged objects look off. I think this is because out focus is drawn to these parts more than other areas of the image that may be aligned better. The squared sum of differences is not biased to any area of the image, but maybe it should be. A further goal could be to adjust the matching metric to weight more important areas of the images like faces or objects in the center. This may help further improve some of these images.
